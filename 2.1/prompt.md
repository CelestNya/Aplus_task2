# 音乐播放器 Web 应用 极致详尽开发规范（最终定稿·直接投喂AI）
## 一、项目核心总规则
### 1. 交付文件（唯一且独立，无任何依赖）
- 前端单文件：`music-player.html`（内嵌**100%原生HTML+CSS+JavaScript**，无外部CSS、无JS库、无CDN、无字体引用）
- 后端单文件：`server.py`（**仅使用Python3标准库**，禁止引入任何第三方包，无pip安装需求）
### 2. 运行流程
1.  将`music-player.html`与`server.py`放在**同一级目录**
2.  终端执行`python server.py`启动HTTP服务，固定监听`8080`端口
3.  浏览器直接打开`music-player.html`，或访问`http://localhost:8080/music-player.html`使用
### 3. 音频文件规则
- 自动扫描**脚本同级目录**下所有`.mp3`格式音频文件
- 不从文件名解析信息，改为**读取MP3文件内置ID3元数据**
- 元数据对应规则：页面**粗体大字=歌曲标题(title)**，**浅灰小字=歌手(artist) · 专辑名(album)**，无元数据时默认显示文件名为歌名、未知歌手/专辑
- 所有文本禁止硬编码，完全从后端读取元数据后动态渲染，确保界面元素没有互相挤压遮挡
### 4. 全局样式禁令
- 禁止任何阴影、渐变动画、过渡动画、鼠标悬浮效果
- 禁止弹性拉伸、响应式变形，所有元素尺寸、间距**固定像素值**
- 禁止滚动条、默认浏览器样式干扰，全局重置默认样式

## 三、前端功能强制实现细则（修复所有已知BUG）
### 3.1 进度条拖动异常修复（核心硬性要求）
1.  技术选型锁定：**仅使用HTML5原生``标签**，禁止引入任何音频第三方库
2.  拖拽逻辑禁令：**绝对禁止通过重设audio.src实现进度跳转**，仅允许修改`audio.currentTime`属性
3.  事件监听规则
    - 监听`mousedown`/`mousemove`/`mouseup`/`mouseleave`全鼠标事件
    - 移动端兼容`touchstart`/`touchmove`/`touchend`触摸事件
    - 拖拽时阻止页面默认滚动行为，`preventDefault()`强制生效
4.  进度同步逻辑
    - 通过`audio.timeupdate`事件**每10ms实时同步**进度条宽度与时间文本
    - 拖拽时暂停自动进度更新，拖拽结束后恢复同步，无冲突、无跳动
    - 拖拽范围严格限制在0~100%，禁止越界，跳转后播放位置精准匹配
5.  时间格式化：统一为`MM:SS`格式，数字不足两位**自动补0**（例：1分3秒→01:03）

### 3.2 上一曲/下一曲状态同步修复（核心硬性要求）
1.  全局状态管理：定义唯一布尔变量`let isPlaying = false`，统一管控播放/暂停状态
2.  切歌行为规则
    - 若当前`isPlaying = true`：切歌后自动加载新音频，调用`audio.play()`，保持播放状态
    - 若当前`isPlaying = false`：切歌后加载新音频，调用`audio.pause()`，保持暂停状态
3.  切歌数据同步
    - 切换后重置`audio.currentTime = 0`，进度条归零
    - 自动更新歌名、歌手、专辑名、总时长文本，无延迟、无错乱
4.  列表边界处理
    - 当前为第一首歌，点击上一曲→跳转至最后一首歌
    - 当前为最后一首歌，点击下一曲→跳转至第一首歌
    - 列表为空时，按钮禁用，显示「暂无音频」提示

### 3.3 基础播放功能细则
1.  播放/暂停切换
    - 点击核心按钮，翻转`isPlaying`状态，同步切换播放/暂停图标
    - 播放时音频正常推进，暂停时冻结当前进度，无异常跳转
2.  音量控制
    - 音量条默认初始值：80%（`audio.volume = 0.8`）
    - 拖拽音量条实时修改`audio.volume`（0~1区间），无延迟、无跳变
    - 音量值与进度填充长度完全匹配
3.  音频列表加载
    - 页面`DOMContentLoaded`后，立即请求后端`/api/list`接口
    - 加载完成后默认选中第一首歌，渲染对应歌名、歌手、专辑名
4.  异常兜底处理
    - 无MP3文件：页面显示「当前目录无音频文件」灰色提示文本
    - 音频加载失败：控制台打印错误，不崩溃页面，保持UI完整
    - 接口请求失败：提示「获取歌曲列表失败」，无样式错乱

## 四、后端 Python 极致规范（仅标准库，无冗余）
### 4.1 依赖与服务配置
1.  仅允许导入标准库：`http.server`、`socketserver.TCPServer`、`os`、`json`、`urllib.parse`
2.  服务配置：固定端口`8080`，绑定`0.0.0.0`，支持局域网访问
3.  代码结构：单文件极简实现，分函数编写（扫描文件、读取MP3元数据、接口处理、静态资源返回）

### 4.2 接口精确定义（无模糊参数）
1.  GET `/api/list`
    - 响应格式：`application/json`，UTF-8编码
    - 返回数据结构：`[{"title":"歌曲名","artist":"歌手","album":"专辑名","file":"文件名.mp3"}]`
    - 扫描逻辑：遍历同级目录，筛选后缀为`.mp3`的文件，读取文件ID3元数据获取标题、歌手、专辑信息；无元数据时title使用文件名，artist与album默认为「未知」
2.  GET `/music/[filename].mp3`
    - 功能：读取对应MP3文件，返回二进制流
    - 响应头：`Content-Type: audio/mpeg`，支持浏览器原生播放
3.  GET `/music-player.html`
    - 功能：读取本地`music-player.html`文件，返回文本流
    - 响应头：`Content-Type: text/html; charset=utf-8`

### 4.3 跨域与兼容处理
1.  所有接口强制添加响应头：
    - `Access-Control-Allow-Origin: *`
    - `Access-Control-Allow-Methods: GET, POST, OPTIONS`
    - `Access-Control-Allow-Headers: Content-Type`
2.  异常捕获
    - 端口占用：提示「端口8080已被占用」，优雅退出
    - 文件不存在：返回404状态码，无程序崩溃
    - 目录读取权限不足：打印错误提示，不阻塞服务启动

### 4.4 代码精简要求
- 无多余注释、无无用变量、无空函数
- 无复杂逻辑封装，仅保留核心功能
- 可直接运行，无语法错误、无缩进问题
