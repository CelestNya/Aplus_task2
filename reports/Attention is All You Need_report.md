# 读 Attention is All You Need （使用豆包伴读，辅以网上搜索到的优质译文）

## 核心贡献

1. 首次构建了完全依赖注意力机制的Transformer模型，摆脱对RNN的时序性依赖
2. 首次使用多头注意力机制，提高了模型的表示能力，可以关注到序列中的不同位置之间的关系
3. 使用位置编码机制，让模型可以并行处理输入序列，降低了模型的计算复杂度

## 方法详解

### 核心方法：**多头自注意力机制**

自注意力：自己与自己进行注意力计算，关注的是序列中的每个位置与其他位置之间的关系
多头：把总维度分成多个子维度，每个子维度分别进行注意力计算，有效捕捉文字间的多种层次关联（如语法、指代、局部、长距离、语义、语序、掩码、细节等）

### 具体过程：经过思考和探索，我大致能理解了Transformer模型的计算具体计算过程
###### 以论文中Transformer完整的计算过程为例（翻译任务）：

#### 两个主要组件

1. MSA：多头自注意力机制
   - 层归一化LayerNorm(X)
   - 输入向量经过三个权重矩阵$W_q$、$W_k$、$W_v$，分别线性变化提取出查询向量Q、键向量K和值向量V。
   - 对Q、K、V进行头分割，每个头的维度为$d_{sub}=\dfrac{d_{model}}{h}$（原文h=8，$d_{model}$=512，$d_{sub}$=64）。
   - 计算注意力分数 $\text{Score}=\dfrac{QK^T}{\sqrt{d_{sub}}}$，并softmax归一化得到注意力权重（权重代表了查询向量与键向量之间的关系强度）。
   - 注意力 $\text{Output}=\text{Score}\times V$，得到最终包含注意力权重的多头值向量。
   - 转置+合并头：将8个子维度的注意力向量重新组织为一个向量，维度为$d_{model}$。
   - 残差连接层：$X_{MSA}=X+\text{Output}$ ，其中$X$为输入向量，相当于把注意力变化叠加到输入向量上，保持模型的表示能力。
  
2. FFN：全连接层
   - 层归一化$LayerNorm(X_{MSA})$
   - 全连接层（512-2048-512），提取高维抽象语义信息（原文隐藏层为2048维），得到512维向量X_{FFN}。
   - 残差连接层：$X_{out}=X_{MSA}+X_{FFN}$
   - 输出最终的源语言语义向量$X_{out}$，形状与输入向量相同。



#### 源语言输入处理
1. 源语言输入转换为token序列，使用源语言embedding成高维向量，同时把位置编码为位置向量，直接叠加得到最终的输入向量。（原文超参数：$d_{model}$=512）

2. 6x 自注意力-全连接层，重复6次有效提取抽象语义信息
   - 进入MSA掩码自注意力
   - 进入FFN提取非线性信息，得到最终源语言语义向量$X_{src}$。
     
#### 目标语言输入处理
1. 目标语言输入，右移后进行相同的词嵌入处理（序列最左侧移添加一个start标签形成初始输入）
2. 进入Masked MSA（掩码在训练时用于防止模型关注未来信息），得到当前序列各个位置之间的关系得到$X_{tgt}$。
3. 进入注意力层，使用输入源语言的语义向量$X_{src}$计算K、V向量以及目标语言输入的$X_{tgt}$计算Q向量，再进行注意力计算，得到目标语言空间的语义向量$X_{tgt}$。
4. 进入全连接层，对$X_{tgt}$进行高维语义表示，在目标语言embedding中进行解码，得到目标语言的下一个token。实现逐词翻译。

## 个人思考

#### 在探索transformer和bert、gpt等衍生结构时，我也了解了不少最新模型的各种优化技巧和SSM原理衍生的Mamba模型，这些模型在计算复杂度和参数量方面都有所优化，为Transformer模型的进一步发展提供了参考。也让我对原版Transformer模型有了评判性能的标准。

1. 模型局限性：
    - 首当其冲的就是复杂度，双向注意力$QK^T$计算复杂度为$O(n^2)$，导致在长序列上计算效率低下。（最新模型通过TopK attention优化让计算复杂度为$O(nk)$，有效提高了模型的计算效率）
    - 注意力分数稀疏（来自ai的观点）：全局softmax使注意力分数稀释，导致模型在长序列上关注到的局部信息减少，影响模型的翻译能力。
    - 推理是串行的(因果依赖)，不像输入可以并行处理，计算效率低下。
    - 每一次计算要大量访存，使计算上限很大程度上受到显存的限制。

2. 数学思想局限性：
    - 论文强行假设了自然语言的词句逻辑可以用两两关注的注意力建模，而自然语言是时序性依赖，哪怕使用了位置编码，也不能完全模拟自然语言的时序性。（为什么大语言模型早期会出现所谓“对时间没有概念”）
    - 对比SSM，transformer模型**没有**全局状态，每一次推理都是独立的，不会受到之前推理的影响。
    - 每一次注意力都是在**穷举**token之间的关系，计算低效。
